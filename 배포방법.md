# 배포 방법 (comcom-as 홈페이지 + 부정클릭 탐지 통합)

이 zip 안의 파일들은 지금 쓰고 있는 comcom-as.vercel.app 프로젝트에 그대로 덮어써서
같은 도메인/같은 배포로 부정클릭 탐지 기능까지 함께 올라가도록 만든 구조입니다.

## 1. Supabase 준비 (ERP와 별도 프로젝트로)

1. supabase.com 에서 **새 프로젝트**를 하나 만듭니다 (ERP에서 쓰는 프로젝트와 절대 공유하지 않기).
2. 프로젝트 대시보드 > SQL Editor 에서 `supabase_schema.sql` 파일 내용을 통째로 붙여넣고 실행합니다.
3. 프로젝트 설정(Settings > API)에서 아래 두 값을 복사해둡니다.
   - Project URL (예: `https://xxxxx.supabase.co`)
   - **service_role** 키 (anon 키 아님! service_role 키를 써야 서버가 자유롭게 읽고 씁니다)

## 2. Vercel 환경변수 등록

comcom-as 프로젝트 > Settings > Environment Variables 에 추가:

| Key | Value |
|---|---|
| `SUPABASE_URL` | 위에서 복사한 Project URL |
| `SUPABASE_SERVICE_KEY` | 위에서 복사한 service_role 키 |

## 3. 파일 올리기

이 zip 안의 내용물(index.html, images/, api/, guard/, tracker.js, requirements.txt, vercel.json) 을
comcom-as 프로젝트 저장소(또는 배포 폴더) 최상위에 그대로 덮어씁니다.

- Git 연동이면: 이 파일들을 저장소에 커밋 + push → Vercel이 자동 재배포
- 직접 업로드라면: 기존 파일 전체를 이 zip 내용물로 교체 후 재배포

## 4. 배포 후 확인할 것

1. `https://comcom-as.vercel.app/` — 홈페이지 정상 노출 + 후기 캐러셀 정상 동작
2. `https://comcom-as.vercel.app/api/stats` — `{"total_clicks":0, ...}` 같은 JSON이 보이면 백엔드 정상
3. `https://comcom-as.vercel.app/guard/` — 부정클릭 탐지 대시보드 화면 노출
4. 홈페이지를 한번 방문해보고 `/api/stats`를 새로고침해서 total_clicks가 1 늘었는지 확인 (tracker.js가 정상 작동 중인지 검증)
5. 홈페이지 우측 하단에 "부정클릭 방지 시스템 작동중" 배지가 항상 떠 있는지 확인
6. 일부러 짧은 시간에 광고 링크를 여러 번 눌러서(3분 이내 3회 이상 등) 경고 배너가 뜨는지 확인

## 5. 의심 IP 처리 (사람이 하는 마지막 단계)

기존 README.md의 안내와 동일합니다.
1. `/guard/` 대시보드에서 의심 IP 확인
2. 체크박스로 선택 → "선택 IP 복사"
3. 네이버 광고관리시스템 > 도구 > 광고노출제한 관리 에 붙여넣어 차단 등록
4. 필요하면 clean.naver.com 에서 "무효클릭 문의하기"로 환불 신청
5. 처리 완료된 IP는 대시보드에서 "차단완료 표시"

## 6. 방문자에게 보이는 경고 시스템 (완료)

- **상시 배지**: 모든 방문자에게 우측 하단에 "부정클릭 방지 시스템 작동중" 배지가 항상 노출됩니다. CCTV 안내판처럼 반복 클릭을 사전에 억제하는 목적입니다.
- **경고 배너**: 짧은 시간 반복 클릭 등 의심 기준에 걸린 방문자에게만 아래 문구의 경고창이 뜹니다.
  - "짧은 시간 동안 광고를 반복적으로 클릭하고 계신 것이 확인되었습니다..."
  - "악의적인 반복 클릭으로 확인될 경우, 사업자 확인을 거쳐 해당 IP는 광고 접속 제한 조치될 수 있습니다."
  - 최근 방문 이력(검색어/시간)과 "즐겨찾기에 추가하기" 버튼도 함께 표시됩니다.
- 전부 `tracker.js` 안에서 자동 처리되므로 별도 설정 없이 그대로 작동합니다.
